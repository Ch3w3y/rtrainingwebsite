What is R?

R is a programming language and free software environment for statistical computing and graphics. It was created by Ross Ihaka and Robert Gentleman at the University of Auckland, New Zealand, and is currently developed by the R Development Core Team.

Key Features of R

  • Statistical Focus: Designed specifically for data analysis and statistics
  • Graphics Capabilities: Excellent tools for data visualization
  • Open Source: Free to use and modify
  • Community Packages: Extensive ecosystem of user-contributed packages
  • Cross-Platform: Runs on Windows, macOS, and Linux

When to Use R

R is particularly useful for:

  • Statistical analysis
  • Data visualization
  • Data cleaning and preparation
  • Machine learning
  • Scientific research

Installing R and RStudio

To get started with R, you'll need to install both R itself and RStudio, a powerful integrated development environment (IDE) for R.

Step 1: Install R

  1. Go to the CRAN (Comprehensive R Archive Network) website
  2. Select your operating system:
    • Windows: Click "Download R for Windows" → "base" → "Download R x.x.x for Windows"
    • macOS: Click "Download R for macOS" and select the appropriate package
    • Linux: Click "Download R for Linux" and follow instructions for your distribution
  3. Run the installer and follow the default installation options

Step 2: Install RStudio

  1. Go to the RStudio Desktop Download page
  2. Click "Download RStudio" for your operating system
  3. Run the installer and follow the default installation options

RStudio Interface

RStudio provides a user-friendly interface divided into four main panes:

The Four Panes

  1. Source Editor (top-left): Where you write and edit your R scripts
  2. Console (bottom-left): Where you can type commands and see results immediately
  3. Environment/History (top-right):
    • Environment tab: Shows all objects currently in memory
    • History tab: List of previously executed commands
  4. Files/Plots/Packages/Help (bottom-right):
    • Files tab: File browser
    • Plots tab: View graphical output
    • Packages tab: Manage installed packages
    • Help tab: Access documentation

Customizing RStudio

You can customize the RStudio interface by going to Tools → Global Options. Some useful settings to adjust:

  • Appearance: Change color themes and font size
  • Code: Adjust code completion and display options
  • Pane Layout: Change the arrangement of the panes

Basic Operations in R

Let's start with some fundamental operations in R.

R as a Calculator

You can use R for basic arithmetic operations:

# Addition
5 + 3  # 8

# Subtraction
10 - 4  # 6

# Multiplication
6 * 7  # 42

# Division
20 / 4  # 5

# Exponentiation
2 ^ 3  # 8 (2 raised to the power of 3)

# Modulo (remainder after division)
17 %% 5  # 2

# Integer division
17 %/% 5  # 3 (how many complete times 5 goes into 17)

Variables and Assignment

You can store values in variables using the assignment operator (<-):

# Assign a value to a variable
x <- 10
y <- 5

# Perform operations with variables
z <- x + y  # z is now 15

# Print a variable's value (several ways)
z           # 15
print(z)    # 15
cat("The value of z is", z, "\n")  # The value of z is 15

# R also allows = for assignment, but <- is preferred
a = 20      # Works, but not recommended
b <- 30     # Preferred style

Variable Naming Rules

When naming variables in R, follow these rules:

  • Names can contain letters, numbers, underscores (_) and periods (.)
  • Names must start with a letter or a period
  • If a name starts with a period, it cannot be followed by a number
  • Names are case-sensitive (age, Age, and AGE are different variables)
# Valid variable names
my_variable <- 10
myVariable <- 20
my.variable <- 30
.hidden_var <- 40

# Invalid variable names (would cause errors)
# 2nd_var <- 50       # Cannot start with a number
# my-variable <- 60   # Cannot contain hyphens
# .2var <- 70         # Cannot start with period followed by number

Working with Packages

R's functionality can be extended with packages, which are collections of functions, data, and documentation.

Installing Packages

You can install packages from CRAN (Comprehensive R Archive Network) using the install.packages() function:

# Install a single package
install.packages("dplyr")

# Install multiple packages at once
install.packages(c("ggplot2", "tidyr", "readr"))

Loading Packages

After installing a package, you need to load it into your current R session to use its functions:

# Load a package
library(dplyr)

# You can now use dplyr functions
# For example, filter() from dplyr
# Without loading the package, you would need to use dplyr::filter()

Checking Installed Packages

You can view all installed packages and their versions:

# List all installed packages
installed.packages()

# More readable way to check specific packages
packageVersion("dplyr")  # Shows version of dplyr package

Updating Packages

It's a good practice to keep your packages up to date:

# Update all packages
update.packages()

# Update specific packages
install.packages("dplyr")  # Reinstalling will update to the latest version

Popular R Packages

Package Description Common Uses
dplyr Data manipulation Filtering, selecting, mutating, summarizing data
ggplot2 Data visualization Creating publication-quality graphics

Getting Help in R

R has a comprehensive built-in help system that can assist you when you're stuck or need information about functions.

Accessing Help Documentation

There are several ways to access help in R:

# Get help for a specific function using ?
?mean
help(mean)  # Alternative syntax

# Get help on a package
help(package = "dplyr")

# Search for help on a topic
??correlation  # Searches for "correlation" in all documentation

Help Page Structure

A typical R help page contains these sections:

  • Description: Brief description of the function
  • Usage: The function's syntax
  • Arguments: Explanation of each parameter
  • Details: Additional information about how the function works
  • Value: What the function returns
  • Examples: Working examples of the function

Function Arguments and Examples

You can quickly check a function's arguments:

# See arguments without full help page
args(mean)  # function (x, ..., na.rm = FALSE, trim = 0)

# View function code (for non-primitive functions)
mean

# Get examples for a function
example(mean)

Online Resources

Beyond R's built-in help, these resources are invaluable:

  • RDocumentation: Searchable interface for R package documentation
  • Stack Overflow: Q&A for programming problems
  • R-bloggers: Collection of R-related blogs
  • rdrr.io: R package documentation, search and code execution

R Scripts

While you can type commands directly in the console, for more complex analyses, you'll want to save your code in R scripts.

Creating and Running Scripts

In RStudio, you can create a new script by clicking File → New File → R Script, or pressing Ctrl+Shift+N (Cmd+Shift+N on Mac).

# This is an example R script

# Set working directory (where your files are located)
setwd("C:/Users/YourName/Documents/R_Projects")  # Windows example
# setwd("/Users/YourName/Documents/R_Projects")  # Mac example

# Load packages
library(dplyr)
library(ggplot2)

# Create some data
x <- 1:10
y <- x^2

# Make a data frame
data <- data.frame(x = x, y = y)

# Print the data
print(data)

# Create a plot
plot(x, y, main = "Square Function", 
     xlab = "x", ylab = "y = x^2", 
     type = "b", col = "blue")

# Save the plot to a file
# pdf("my_plot.pdf")
# plot(x, y)
# dev.off()

Running Code from Scripts

You can execute code from your script in several ways:

  • Run the current line: Ctrl+Enter (Cmd+Enter on Mac)
  • Run selected lines: Select code + Ctrl+Enter
  • Run the entire script: Ctrl+Shift+Enter or click the "Source" button

Script Best Practices

  • Comments: Use # to add comments explaining your code
  • Organization: Group related code together with clear section headers
  • Documentation: Document data sources, assumptions, and goals at the top
  • Readability: Use consistent indentation and spacing
  • Modularity: Break complex tasks into smaller, manageable functions
