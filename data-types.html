Basic Data Types in R

R has several basic data types that are fundamental to understanding how data is represented and manipulated:

Numeric

Numeric values are the default computational data type in R. They include both integers and double-precision floating-point numbers.

# Numeric examples
x <- 42     # Integer
y <- 3.14   # Decimal
z <- 2e3    # Scientific notation (2000)

# Check the type
class(x)  # "numeric"
class(y)  # "numeric"
class(z)  # "numeric"

# Force an integer type
a <- 42L   # The L suffix creates an integer
class(a)  # "integer"

Character

Character data types store text values. In R, character strings can be created using either single or double quotes.

# Character examples
name <- "Alice"
greeting <- 'Hello, world!'

# Character operations
nchar(name)      # 5 (number of characters)
paste(greeting, name)  # "Hello, world! Alice"
toupper(name)    # "ALICE"

Logical (Boolean)

Logical values represent true/false conditions. R uses TRUE and FALSE as built-in constants.

# Logical examples
x <- TRUE
y <- FALSE

# Logical operations
!x         # NOT: FALSE
x & y      # AND: FALSE
x | y      # OR: TRUE

# Logical results from comparisons
5 > 3      # TRUE
10 == 10   # TRUE
"a" == "b" # FALSE

Vectors

Vectors are the most basic data structure in R. They are one-dimensional arrays that can hold data of the same type.

# Creating vectors
numeric_vector <- c(1, 2, 3, 4, 5)
character_vector <- c("apple", "banana", "cherry")
logical_vector <- c(TRUE, FALSE, TRUE, TRUE)

# Vector operations
numeric_vector + 10         # Add 10 to each element
numeric_vector * 2          # Multiply each element by 2
length(character_vector)    # 3 (number of elements)
sum(logical_vector)         # 3 (TRUE counts as 1, FALSE as 0)

# Vector indexing (1-based, not 0-based)
numeric_vector[1]           # 1 (first element)
character_vector[2:3]       # c("banana", "cherry") (elements 2 through 3)
logical_vector[c(1, 4)]     # c(TRUE, TRUE) (first and fourth elements)

# Negative indexing excludes elements
numeric_vector[-3]          # c(1, 2, 4, 5) (all except the third element)

Vector Recycling

When performing operations with vectors of different lengths, R will recycle the shorter vector to match the length of the longer one.

# Vector recycling example
x <- c(1, 2, 3, 4, 5)
y <- c(10, 20)

# y will be recycled to c(10, 20, 10, 20, 10)
x + y   # c(11, 22, 13, 24, 15)

Missing Values in R

R has two special values for representing missing or undefined data: NA and NULL.

NA (Not Available)

NA represents a missing value or a value that is not available. NAs are contagious in calculations.

# Working with NA values
x <- c(1, 2, NA, 4, 5)

# NA propagates in calculations
sum(x)               # NA
mean(x)              # NA

# Use na.rm parameter to remove NAs in calculations
sum(x, na.rm = TRUE) # 12
mean(x, na.rm = TRUE) # 3

# Check for NA values
is.na(x)             # c(FALSE, FALSE, TRUE, FALSE, FALSE)
any(is.na(x))        # TRUE
sum(is.na(x))        # 1 (count of NA values)

NULL

NULL represents the absence of a value or an undefined value. Unlike NA, NULL has no type and length zero.

# Working with NULL
y <- NULL

# NULL behaves differently than NA
length(y)           # 0
typeof(y)           # "NULL"

# NULL in vectors
c(1, 2, NULL, 4)    # c(1, 2, 4) - NULL is removed

# Checking for NULL
is.null(y)          # TRUE
is.null(NA)         # FALSE

Key Differences between NA and NULL

Feature NA NULL
Represents Missing value Absence of value
Has type Yes (NA_integer_, NA_character_, etc.) No (NULL is its own type)
Length 1 0
In vectors Takes up space Disappears
Detection is.na() is.null()
